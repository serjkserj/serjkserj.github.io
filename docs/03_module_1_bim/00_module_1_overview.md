# Модуль 1. Что такое BIM на самом деле

## Зачем нужен этот модуль

После модуля 0 у читателя уже есть базовая карта проектной среды. Но этого еще недостаточно, чтобы понять, почему вокруг BIM так много разговоров и почему обычной “трехмерной модели” для современной работы уже мало.

Этот модуль нужен для того, чтобы заменить поверхностное представление о BIM на более зрелую и практическую логику.

## Главная задача модуля

Задача модуля — показать, что BIM:

- не равен красивой 3D-модели;
- не сводится к названию программы;
- не существует отдельно от процессов и требований;
- важен потому, что помогает работать с моделью как с источником данных.

## Что будет дальше

После этого модуля читатель должен начать видеть BIM не как модное слово, а как рабочую систему.

Это означает, что дальше будет легче понять:

```text
BIM -> модель как данные -> Revit -> IFC -> проверки -> АГР / экспертиза
```

Именно этот переход нужен, чтобы затем осмысленно войти в роль BIM-координатора, а не просто выучить еще несколько терминов.
