# Карточки документов для ссылочного аппарата

Это reader-facing слой ссылочного аппарата.

Он нужен для двух задач:

- давать читателю нормальную карточку документа вместо сырого пути к файлу;
- держать в одном месте краткий реестр источников, на которые опираются сноски handbook.

## Как читать карточки

У каждой карточки есть:

- `source_id`;
- полное название;
- короткое название для текста;
- тема;
- происхождение;
- статус;
- ключевые даты;
- как именно документ используется в handbook.

## Короткий вывод

Именно для этого и нужен этот блок.
