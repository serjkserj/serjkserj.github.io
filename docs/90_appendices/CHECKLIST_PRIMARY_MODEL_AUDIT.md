# CHECKLIST PRIMARY MODEL AUDIT

Этот чек-лист — производный стартовый инструмент для первичной диагностики. Он не заменяет требования конкретного рубежа и должен читаться вместе с документами, которые реально управляют текущим пакетом и проверкой.[^appendix-primary-audit]

## 1. Общая ориентировка

- [ ] Понятна стадия проекта
- [ ] Понятна цель текущей проверки
- [ ] Понятно, какая модель является основной
- [ ] Понятен ближайший внешний рубеж или формальный пакет передачи
- [ ] Понятно, на какой документ, приложение или правило проверки мы опираемся

## 2. Структура модели

- [ ] Уровни и этажность читаются последовательно
- [ ] Нет явного структурного хаоса в элементах и пространствах
- [ ] Понятно, как организованы ключевые сущности модели

## 3. Геометрия

- [ ] Нет очевидных критичных геометрических конфликтов
- [ ] Архитектурные элементы собраны последовательно
- [ ] Критичные места помечены для более детальной проверки

## 4. Данные

- [ ] Ключевые параметры не пусты без причины
- [ ] Нет очевидного конфликта между геометрией и данными
- [ ] Понятны слабые места атрибутивного слоя
- [ ] Не смешаны обязательное ядро данных, методика расчета и внутренние служебные надстройки

## 5. Пространства и показатели

- [ ] Помещения и зоны читаются осмысленно
- [ ] Площади и показатели не выглядят случайными
- [ ] Понятно, где искать источник расчетных значений

## 6. Итог

- [ ] Сформулирован список критичных проблем
- [ ] Отделены блокеры от второстепенных замечаний
- [ ] Понятен следующий шаг после аудита
- [ ] Понятно, какие проблемы относятся к модели, а какие к пакету, приложению или правилу передачи

Лучше всего этот чек-лист работает как быстрый старт к модулю про аудит модели и к практической работе BIM-координатора на проекте.
