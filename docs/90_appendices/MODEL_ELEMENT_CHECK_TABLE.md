# MODEL ELEMENT CHECK TABLE

| Элемент / сущность | Что проверить в первую очередь | Типовой риск |
|---|---|---|
| Уровни и этажи | Последовательность, читаемость, отсутствие логических разрывов | Путается этажная структура и дальнейшая проверка |
| Стены | Роль элемента, базовые данные, читаемость в модели и IFC | Объект виден визуально, но плохо читается как сущность |
| Перекрытия | Структурная роль, согласованность с уровнями | Проблемы этажности и координации |
| Двери и окна | Привязка, типизация, базовые атрибуты | Ошибки в фасадной и архитектурной логике |
| Помещения | Наличие, названия, логика использования | Пустые или противоречивые пространства |
| Зоны | Связь с задачей проекта, показатели, атрибуты | Слабая расчетная логика |
| Участок / площадка | Базовые параметры и привязка к внешней логике | Разрыв между моделью и требованиями к подаче |
| IFC-файл после экспорта | Общая читаемость, ключевые сущности, критичные потери | Ошибки обменного представления |
