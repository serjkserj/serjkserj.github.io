# CHECKLIST PREPARE EXPERTISE

## 1. Структура и читаемость модели

- [ ] Модель читается как система, а не как авторский файл
- [ ] Основные сущности распознаются однозначно
- [ ] Нет критичного структурного хаоса
- [ ] Состав передаваемого пакета собран осознанно
- [ ] Именование файлов и цифровых частей модели не выглядит случайным
- [ ] Уровни, оси и пространственная структура читаются вне команды авторов
- [ ] Если процедура требует не только `IFC`, но и другой формат вроде `LandXML`, это учтено заранее

## 2. Негеометрический слой

- [ ] Ключевые параметры заполнены
- [ ] Классификация и атрибуты не противоречат друг другу
- [ ] Пространства, зоны и показатели согласованы
- [ ] Элементы идентифицированы достаточно устойчиво для внешнего чтения
- [ ] Экспортируемые атрибуты не смешаны с чисто внутренними служебными полями
- [ ] Таблицы атрибутов читаются по сущностям, а не как бесформенный набор полей
- [ ] Если используются `Pset_*`, `RusSet_*` или `RUS_*`, их роль для внешнего чтения понятна

## 3. Междисциплинарная логика

- [ ] Известные междисциплинарные конфликты разобраны
- [ ] Понятно, где риск локальный, а где системный
- [ ] Команда не проверяет архитектурную модель в отрыве от смежников
- [ ] Координация между моделями и разделами не оставлена на устные пояснения

## 4. Формальный контур проверки

- [ ] Формат и структура пакета помогают внешнему чтению
- [ ] Модель не выдает неполный или случайный экспорт за официальный пакет
- [ ] Если для процедуры важна юридическая значимость файлов, это учтено в составе передачи
- [ ] Критичные коллизии разобраны, а не перенесены "на потом"
- [ ] Аннотации и замечания можно использовать как часть повторяемого контроля
- [ ] Есть понимание, какой пункт относится к основной части документа, а какой спрятан в приложении или таблице

## 5. Финальная готовность

- [ ] Модель готова к более формальному чтению
- [ ] Повторяющиеся слабые места учтены заранее
- [ ] Команда понимает реальные риски перед передачей
- [ ] Понятно, какие проверки относятся к общим требованиям, а какие — к специализированному типу объекта

Этот чек-лист не должен заменять чек-лист к АГР: его полезно использовать именно как отдельную рамку для более строгого внешнего чтения модели.
