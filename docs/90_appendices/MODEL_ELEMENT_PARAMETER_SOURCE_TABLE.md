# MODEL ELEMENT PARAMETER SOURCE TABLE

Эта таблица — производная карта соответствий. Она собирает типовые связки между сущностью, внешним атрибутивным слоем и источником значения, а не заменяет карточки документов и карту параметров проекта.[^appendix-param-source-table]

| Элемент / сущность | Внешний слой | Типичный набор полей | Типичный источник значения | Кто чаще отвечает |
|---|---|---|---|---|
| Помещение / `IfcSpace` | Пространства, расчеты, экспертиза | Наименование, назначение, `Pset_SpaceCommon`, часть `Pset_SpaceFireSafetyRequirements` | Архитектурное решение, таблицы помещений, логика проекта | Архитектор / BIM-координатор контролирует |
| Зона | Расчетный слой, `АГР`, `ТЭП` | Тип зоны, расчетные признаки, показатели | Расчетная логика проекта, таблицы, требования к `АГР` | Архитектор / BIM-координатор |
| Участок / `IfcSite` | Внешний контур `АГР` и участка | Базовые идентификационные и площадные данные, `RusSet_AGR`, `RUS_LandID`, `RUS_AreaSquare`, `RUS_Area` | Исходные документы, расчеты, проектная ситуация | Проектировщик / координатор контролирует |
| Архитектурный элемент / `IfcWall`, `IfcDoor`, `IfcWindow` | Классификация и базовые свойства | Сущность, код, `Pset_*`, примеры `LoadBearing`, `FireRating`, `Compartmentation`, `IsExternal` | Проектное решение, карта параметров, классификатор | Архитектор / BIM-координатор |
| IFC-атрибут / `Pset_*`, `RusSet_*`, `RUS_*` | Значение для обмена и проверки | Внешний атрибут, проектный аналог, тип данных, правило заполнения | Параметры модели, правила экспорта, карта соответствия | BIM-координатор / BIM-специалист |
| Показатель / `ТЭП` | Расчетный выход модели | Площадь, объем, группировка, источник расчета, `XML`-выход | Зоны, площади, таблицы расчета, расчетная логика проекта | Архитектор / BIM-координатор / расчетная логика проекта |

Примечание:
- внутренние поля бюро и `ATR*`-параметры не должны автоматически попадать в это базовое ядро как универсальная норма.
- рабочая карта параметров должна явно разделять внешний атрибут, проектное поле и внутреннюю подсказку по заполнению.
- при работе с таблицей полезно отдельно помечать, что из этого является обязательным требованием, что относится к методике расчета или заполнения, а что является локальной проектной реализацией.
