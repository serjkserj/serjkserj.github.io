# Модуль 4. IFC с нуля и без мистики

Этот модуль переводит мышление читателя из логики авторской модели в логику обменной модели. Именно здесь становится видно, почему хорошая модель в `Revit` еще не гарантирует хорошую модель в `IFC`.

## Зачем нужен этот модуль

Для новичка `IFC` часто выглядит как пугающая техническая зона “для кого-то другого”. На практике это один из центральных рабочих слоев для `BIM`-координатора, потому что через него проходят:

- обмен;
- проверка;
- автоматизация;
- внешнее чтение модели.

Модуль нужен, чтобы снять мистику и собрать спокойную рабочую опору.

## Как читать этот модуль

Здесь полезно идти по нарастающей.

1. Сначала понять, что такое `IFC` и зачем он вообще нужен.
2. Потом разобрать внутреннюю логику `IFC`-модели.
3. После этого перейти к основным классам и типам сущностей.
4. В конце разобрать мышление в логике `IFC`, экспорт и типовые сбои на выходе.
